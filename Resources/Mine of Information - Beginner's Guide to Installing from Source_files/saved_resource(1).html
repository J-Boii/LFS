                                      Simon Kitching Mod VENUGOPAL K V K • 5 years ago

                                      (sorry my reply is so late..)
                                      Is it really "heck easy" to install software from the sourcecode under Windows? It has been a long time since I last tried, but then it was far less convenient than under Linux.

                                      To install pre-compiled software on Linux is IMO far easier than in Windows - just click the "start menu" and open the "software manager". Then browse through the selection of thousands of available programs, click on the desired ones and choose "install". The programs are all guaranteed to be virus-free (unlike running a "windows installer" downloaded from some site on the internet). And under Linux, the applications automatically get security-patches and bug-fixes.

                                      This article is specifically about installing applications "from source" - to handle odd and unusual software which is not normally available, or special versions of the software (eg to get the very latest features), or to use unusual options (eg leave features out which are normally included, in order to save space or improve security). Or when using something like Linux From Scratch, which is specifically meant as either a learning-tool or for building an extremely specialized and optimised version of Linux. Most computer users don't need any of this, and so can just install pre-compiled software with the easy approach described above.

                                                    Simon Kitching Mod phantom_mullet • 4 years ago • edited

                                                    In general, yes you do need to keep track of what software you installed where - and yes, it is a nuisance. That's one of the big benefits of a package-manager.

                                                    There is an old solution to this problem - the pkg-config program ( https://en.wikipedia.org/wi... ). Every tool that installs software _should_ create a ".pc" file in a common directory (eg /usr/share/pkgconfig); that file then includes information about the app. In particular, it includes info needed when compiling applications from source on that machine which need to interact with pre-installed software (eg link to libs or use the same default config-dirs). It appears that this approach has mostly died out though - on my machine there are only a dozen or so files in /usr/share/pkgconfig.
